Telegram Bot SDK is a TypeScript toolkit for working with the [Telegram Bot API](https://core.telegram.org/bots/api). It consists of three small packages that can be used independently or together — from raw type definitions to a full bot framework.

## Packages

| Package                    | npm                                                                                                                     | What it does                                       |
| -------------------------- | ----------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| `@effect-ak/tg-bot-client` | [![npm](https://img.shields.io/npm/v/@effect-ak/tg-bot-client)](https://www.npmjs.com/package/@effect-ak/tg-bot-client) | Type-safe client for calling Bot API methods       |
| `@effect-ak/tg-bot`        | [![npm](https://img.shields.io/npm/v/@effect-ak/tg-bot)](https://www.npmjs.com/package/@effect-ak/tg-bot)               | Bot framework with handlers, polling, and webhooks |
| `@effect-ak/tg-bot-api`    | [![npm](https://img.shields.io/npm/v/@effect-ak/tg-bot-api)](https://www.npmjs.com/package/@effect-ak/tg-bot-api)       | TypeScript types for the entire Bot API            |

Each package builds on the one below it: the bot framework includes the client, and the client includes the types. Install the client on its own for simpler use cases — sending notifications, managing a channel, integrating Telegram into an existing app — and the framework when you're building a full bot.

## Why this SDK

- **Always up to date** — types are generated from the official Telegram API documentation, not maintained by hand. New Bot API release → re-run the generator → new types.
- **Fully type-safe** — every method knows its parameters and return type, every error is a tagged union you can pattern-match on.
- **Zero runtime dependencies** — native `fetch` and nothing else. Runs on Node.js 18+, Bun, Deno, browsers, and edge runtimes.
- **No webhooks required** — long polling works from any machine without a public URL. Webhooks are there when you need them for serverless.
